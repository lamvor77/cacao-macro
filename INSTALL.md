# 설치 안내 (INSTALL.md)

이 문서는 `cacao_macro`를 새 Windows PC에 설치하고 처음 실행하는 절차를
설명한다. 대상 독자는 프로그램을 배포/설치하는 관리자다.

## 1. 시스템 요구사항

- Windows 10 또는 Windows 11 (64bit)
- 카카오톡 PC버전 설치 및 로그인 상태
- 인터넷 연결(Google 로그인/클라우드 동기화를 쓰는 경우에만 필요 — 오프라인
  전용으로도 동작한다)
- 별도의 Python 설치는 필요 없다(EXE에 모두 포함되어 있음)

## 2. 설치 폴더 준비

임의의 폴더(예: `C:\cacao_macro\`)를 만든다. 한글/공백이 포함된 경로도
동작하지만, 문제가 생겼을 때 경로를 알려주기 쉽도록 영문 경로를 권장한다.

## 3. 파일 배치

배포받은 압축을 풀면 아래 파일들이 들어있다. **이 파일들은 모두 같은
폴더**(2단계에서 만든 폴더)에 있어야 한다.

```
cacao_macro.exe
.env.example   (아래 4단계에서 .env로 바꿔야 한다)
license_build.json
README.txt
```

## 4. .env 설정

1. `.env.example` 파일을 복사해 이름을 `.env`로 바꾼다(같은 폴더 안에).
2. 배포 담당자에게서 받은 실제 값을 채운다. 최종 사용자 배포본에는 보통
   `LICENSE_SECRET_KEY`와 Supabase 관련 값만 필요하다(관리자 전용 값인
   `LICENSE_ADMIN_PASSWORD`는 넣지 않는다).
3. 저장한다. 이 파일은 절대 다른 사람과 공유하지 않는다(비밀값 포함).

## 5. license_build.json 배치

배포 담당자에게서 받은 `license_build.json`을 `.env`와 같은 폴더에 둔다.
사용 기간이 지나면 이 파일만 새로 받아 교체하면 된다(프로그램을 다시 설치할
필요 없음) — 자세한 절차는 `docs/license_deployment_guide.md` 참고.

## 6. 첫 실행

`cacao_macro.exe`를 더블클릭한다.

- 라이선스 파일이 올바르면 프로그램 창이 뜬다.
- 오류가 나면 [문제 해결](#9-windows-보안-경고-대응)과 `README.txt`의
  "라이선스 오류 대응" 항목을 참고한다.

## 7. Google 로그인 (클라우드 동기화를 쓰는 경우)

1. 프로그램 오른쪽 위 "Google로 로그인" 버튼을 누른다.
2. 기본 브라우저가 열리며 Google 로그인/동의 화면이 뜬다.
3. 로그인을 완료하면 브라우저 탭은 닫아도 되고, 프로그램 창으로 돌아온다.
4. 관리자 승인이 필요한 경우, 승인 전까지는 "승인 대기" 상태로 표시된다
   (읽기/쓰기 제한 — 관리자에게 승인을 요청한다).

로그인하지 않아도 프로그램은 완전히 동작한다(로컬 저장/발송 전용).

## 8. 바로가기 생성

`cacao_macro.exe`를 마우스 오른쪽 버튼으로 클릭 → "바로가기 만들기" → 만들어진
바로가기를 바탕화면이나 시작 메뉴로 옮긴다. 바로가기로 실행해도 프로그램은
항상 실제 exe 파일이 있는 폴더를 기준으로 `.env`/`license_build.json`을
찾으므로(실행 위치와 무관), 바로가기 위치는 어디든 상관없다.

## 9. Windows 보안 경고 대응

서명되지 않은 EXE라 Windows Defender SmartScreen이 "Windows의 PC 보호"
경고를 띄울 수 있다. "추가 정보" → "실행"을 눌러 진행한다(배포 담당자가
신뢰할 수 있는 출처에서 받은 파일인 경우에만 진행할 것).

## 10. 프로그램 제거

설치 폴더를 통째로 삭제하면 된다(레지스트리를 사용하지 않는다). 삭제 전
`storage/` 폴더에 남기고 싶은 데이터가 있으면 미리 백업한다(아래 12단계
참고 — 프로그램 안에서도 백업 기능을 제공한다).

## 11. 사용자 데이터 위치

모두 `cacao_macro.exe`와 같은 폴더 아래에 있다(레지스트리나 사용자 프로필
폴더를 쓰지 않는다).

| 위치 | 내용 |
|---|---|
| `storage/` | 저장한 단톡방 목록/메시지(JSON) |
| `storage/cloud_sync/` | 클라우드 동기화 캐시 + 로그인 세션(암호화됨) |
| `logs/` | 날짜별 실행 로그 |
| `backup/` | 자동/수동 백업(ZIP) |

## 12. 데이터 백업

프로그램 오른쪽 위 "진단정보" → "백업 관리..."에서 언제든 수동 백업을 만들
수 있다. 프로그램은 하루 최대 1회 자동 백업도 만든다(종료 시점 기준). 자세한
내용은 `docs/operations_guide.md`의 "백업·복구" 항목을 참고한다.

## 13. 모바일에서 메시지 수정하기

PC 프로그램과는 별도로, 직원들은 모바일 웹(PWA)에서 1~12번 메시지의 내용을
조회/수정할 수 있다. 모바일에서는 카카오톡 발송을 하지 않는다 — 발송은 항상
이 PC 프로그램에서만 수행한다. 모바일 접속 주소, 로그인 방법, 직원 계정 승인
절차는 `docs/mobile_message_editor.md`를 참고한다.
