# 모바일 메시지 편집 (Mobile 실시간 동기화 스프린트)

내부 직원용 모바일 웹(PWA)에서 1~12번 메시지를 조회/수정하는 방법을 설명한다.
대상 독자는 일반 직원과 승인 관리자 둘 다다.

## 이 기능이 하는 일 / 하지 않는 일

**한다:**
- 모바일에서 1~12번 메시지의 제목/내용을 확인하고 수정한다.
- PC에서 수정하면 모바일에, 모바일에서 수정하면 PC(실행 중인 프로그램)에
  거의 즉시 반영된다(Supabase Realtime).

**절대 하지 않는다:**
- 모바일에서 카카오톡으로 직접 발송하지 않는다.
- 모바일에서 메시지를 저장해도 자동으로 카카오톡에 발송되지 않는다.
- 카카오톡 발송은 항상 PC 프로그램에서, 기존 방식 그대로만 수행한다.
- 모바일에서 새 메시지 번호(13번 이상)를 만들 수 없다.
- 모바일에서 메시지를 삭제할 수 없다(항상 1~12번 중 하나를 수정할 뿐이다).

## 1. 모바일 접속 방법

1. 관리자에게 모바일 웹 주소(URL)를 받는다 — 배포 방법은 이 문서 "배포 방법"
   절 참고. 사내에서만 접근 가능하도록 배포하는 것을 권장한다.
2. 스마트폰 브라우저(Chrome/Safari)로 접속한다.
3. (선택) 브라우저 메뉴에서 "홈 화면에 추가"를 누르면 앱처럼 아이콘으로 쓸 수
   있다(PWA).
4. "Google로 로그인" 버튼으로 회사 Google 계정 로그인한다.

## 2. 직원 계정 승인 방법

모바일은 PC 프로그램과 **완전히 같은** 승인 체계(`app_users` 테이블)를 쓴다 —
모바일 전용 별도 가입 절차가 없다.

- 처음 로그인하면 자동으로 `pending`(승인 대기) 상태의 계정이 만들어진다.
- 관리자가 PC 프로그램의 "운영 관리자" 화면에서 해당 계정을 승인(approve)하고
  역할을 `editor` 또는 `admin`으로 지정해야 모바일에서 메시지를 수정할 수 있다.
  - `editor` = 이 문서에서 "employee"라고 부르는 일반 편집 권한.
  - `admin` = 전체 조회/수정/이력 확인 + 강제 저장(충돌 무시) + 초기 마이그레이션 권한.
  - `viewer`(조회 전용) 역할은 이번 스프린트에서는 모바일 접근 권한이 없다
    (모바일은 "조회+수정"이 목적이라 조회 전용 화면은 만들지 않았다).
- 승인 전(`pending`)이거나 차단(`blocked`)된 계정은 로그인은 되지만 "접근할
  수 없습니다" 화면만 보인다 — 메시지 데이터를 전혀 조회할 수 없다.

## 3. 1~12번 메시지 수정 방법

1. 로그인 후 메시지 목록 화면에서 원하는 번호를 누른다.
2. 제목(선택)/내용을 수정한다.
3. "저장"을 누른다.
   - 성공하면 목록으로 돌아간다. 이 내용은 즉시 Supabase에 저장되고, PC
     프로그램이 실행 중이면 그 화면에도 반영된다.
   - 다른 사람이 먼저 저장해 충돌이 나면 "다른 사용자가 먼저 저장했습니다"
     오류가 뜬다 — 최신 내용을 확인한 뒤 다시 수정해서 저장한다.
4. "취소"를 누르면 수정 내용을 버리고 목록으로 돌아간다(저장되지 않음).
5. "최신 내용 다시 불러오기"를 누르면 지금까지 입력한 내용을 버리고 서버의
   최신 값으로 다시 채운다.

편집 중에 다른 직원이 같은 메시지를 수정하면 화면에 "다른 직원이 이 메시지를
수정했습니다" 배너가 뜬다 — 입력창 내용이 조용히 바뀌지 않는다. 세 가지 중
선택한다: 최신 내용 확인 / 현재 작성 내용 유지 / 작성 내용 복사 후 최신
불러오기.

## 4. 배포 방법

`mobile/`은 정적 파일로 빌드되는 일반 Vite 프로젝트다 — Vercel, Netlify,
Cloudflare Pages, 사내 정적 파일 서버 등 어디든 배포할 수 있다(이 프로젝트는
특정 호스팅을 강제하지 않는다).

```powershell
cd mobile
npm install
npm run build
# dist/ 폴더를 정적 호스팅에 업로드
```

배포 전 `.env.example`을 `.env`로 복사해 값을 채운다:

```
VITE_SUPABASE_URL=
VITE_SUPABASE_ANON_KEY=
```

`VITE_SUPABASE_ANON_KEY`는 브라우저에 그대로 노출되는 값이다(anon key는
원래 공개되어도 안전하도록 설계된 키다 — 실제 접근 제어는 Supabase RLS가
한다). **anon key 이외의 값(`service_role`, `LICENSE_SECRET_KEY`, Google
client secret, 운영 관리자 비밀번호)은 이 프로젝트 어디에도 넣지 않는다.**

Supabase 대시보드의 Authentication → URL Configuration에 배포한 모바일
주소를 Redirect URL로 등록해야 Google 로그인이 정상 동작한다.

## 5. 로컬 개발 방법

```powershell
cd mobile
npm install
npm run dev
```

`http://localhost:5173`에서 확인한다. 개발 중에도 실제 Supabase 프로젝트에
연결되므로(anon key 기준), 테스트용 Supabase 프로젝트를 별도로 쓰는 것을
권장한다.
