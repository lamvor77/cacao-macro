# 카카오톡 단톡방 자동 메시지 전송 프로그램

## 프로젝트 목적

본 프로젝트는 Windows 환경에서 실행되는 카카오톡 PC버전 자동 메시지 전송 프로그램이다.

사용자가 선택한 단톡방에 모집글, 안내문, 홍보글 등을 자동으로 반복 전송하는 업무용 프로그램을 개발한다.

실제 업무에서 사용하는 프로그램이므로 안정성과 유지보수성을 가장 중요하게 생각한다.

---

# 개발 원칙

항상 다음 원칙을 따른다.

1. 기능보다 안정성을 우선한다.
2. MVP부터 완성하고 기능을 추가한다.
3. 한 번에 큰 수정보다 작은 단위로 개발한다.
4. 모든 기능은 모듈화한다.
5. 유지보수가 쉬운 구조를 만든다.
6. 코드에는 충분한 주석을 작성한다.
7. 기존 기능을 수정할 때는 다른 기능이 깨지지 않도록 한다.
8. 예외처리를 반드시 구현한다.
9. UI와 로직을 분리한다.
10. 하드코딩을 최소화한다.

# UI 원칙

GUI는 CustomTkinter를 사용한다.

항상 Windows 기본 다크모드와 잘 어울리는 디자인으로 제작한다.

버튼 위치와 크기는 일정하게 유지한다.

스크롤 가능한 영역은 모두 마우스 휠을 지원한다.

---

# 단톡방 관리

왼쪽 목록은 항상 비어있는 상태로 시작한다.

다음 경우에만 목록이 표시된다.

* 활성화된 카톡방 가져오기
* 저장된 목록 불러오기

체크된 단톡방만 발송 대상이다.

---

# 저장 기능

JSON 파일을 사용한다.

저장 항목

* 단톡방 이름
* 체크 상태
* 메시지 1~12

프로그램 시작 시 자동 복원한다.

---

# 저장된 카톡방 켜기

반드시 구현한다.

동작 순서

1. 카카오톡 실행 확인
2. 저장된 단톡방 검색
3. 채팅방 열기
4. 다음 채팅방 반복

이미 열려있는 채팅방은 다시 열지 않는다.

# 가장 중요한 규칙

절대로

```
메시지1 → 전체 단톡방

메시지2 → 전체 단톡방

메시지3 → 전체 단톡방
```

방식으로 구현하지 않는다.

반드시

```
단톡방1

↓

메시지1

↓

2초

↓

메시지2

↓

2초

↓

메시지3

↓

2초

↓

단톡방2

↓

메시지1

↓

2초

↓

메시지2

↓

2초

↓

메시지3
```

순서로 구현한다.

# 로그

모든 작업은 로그를 남긴다.

예)

```

메시지2 완료

메시지3 완료

간병정보2

...

GroupA 완료
```

오류도 반드시 로그를 남긴다.

# 코딩 스타일

* 함수는 하나의 역할만 수행한다.

* 상수는 config로 분리한다.
* UI 코드와 비즈니스 로직을 섞지 않는다.
* 반복 코드는 함수로 만든다.
* 타입 힌트를 사용한다.

# Claude 작업 규칙

새로운 기능을 추가하기 전에 기존 기능이 정상 동작하는지 확인한다.

코드를 수정할 때는 변경된 파일 전체를 보여준다.

임의로 프로젝트 구조를 변경하지 않는다.

사용자의 승인 없이 핵심 로직을 변경하지 않는다.

버그를 수정할 때는 수정 이유를 함께 설명한다.

모든 코드는 실제 실행 가능한 상태로 작성한다.
